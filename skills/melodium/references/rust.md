# Implementing Mélodium Elements in Rust

This guide covers everything needed to write Rust code that exposes Mélodium treatments, functions, models, data types, and contexts using the `melodium_macro` crate.

---

## Crate setup

Every Rust crate that exposes Mélodium elements must call `mel_package!()` at the crate root. This registers the package with the Mélodium runtime.

```rust
// src/lib.rs
use melodium_macro::mel_package;

pub mod my_module;

mel_package!();
```

The crate must also depend on `melodium_core` and `melodium_macro`. Typical imports at the top of each element file:

```rust
use melodium_core::*;           // Value, TransmissionValue, GetData, Data, etc.
use melodium_core::executive::*;
use melodium_macro::{check, mel_treatment, mel_function, mel_data, mel_model, mel_context};
```

---

## `#[mel_treatment]` — defining a treatment

A treatment is an `async fn` that receives data through named input and output objects injected by the macro. The function signature contains only the **treatment parameters** (Mélodium `var`/`const` arguments); inputs and outputs are declared in the macro attributes and become local variables inside the function body.

### Macro syntax

```rust
#[mel_treatment(
    generic T (TraitA + TraitB)   // generic type parameter with optional trait bounds
    generic U ()                  // unconstrained generic
    model my_model ModelType      // model configuration parameter
    input  port_name  Stream<T>   // input port
    input  other_port Block<U>    // Block input
    output out_name   Stream<T>   // output port
    output flag       Block<void> // Block output
)]
pub async fn treatment_name(param: SomeType, other: AnotherType) {
    // body
}
```

- **`generic N (Trait1 + Trait2)`** — declares a generic type `N` constrained by the listed traits. Use `()` for unconstrained.
- **`model name ModelType`** — injects a model handle. Inside the body it becomes an object of type `ModelTypeModel` (generated by `mel_model`), accessed via `ModelTypeModel::into(name)`.
- **`input name Kind<Type>`** / **`output name Kind<Type>`** — ports become local bindings of type `InputPort` / `OutputPort`.
- **Treatment parameters** go on the `fn` signature, not in the macro. They are Mélodium `var` parameters of the treatment.

### Mélodium name vs. Rust name

The Mélodium-facing name is derived from the Rust function name, converted to `camelCase`. `snake_case` Rust names become `camelCase` in Mélodium:

| Rust function name | Mélodium element name |
|--------------------|----------------------|
| `to_utf8` | `toUtf8` |
| `unwrap_or` | `unwrapOr` |
| `generate_indefinitely` | `generateIndefinitely` |
| `log_stream_label` | `logStreamLabel` |

### Receiving data from inputs

```rust
// Stream input — receive a batch of values
while let Ok(values) = input_port.recv_many().await {
    // values: TransmissionValue
}

// Block input — receive exactly one value
if let Ok(val) = input_port.recv_one().await {
    // val: Value
}

// With typed extraction
while let Ok(items) = input_port
    .recv_many()
    .await
    .map(|v| TryInto::<Vec<string>>::try_into(v).unwrap())
{
    for item in items { /* item: String */ }
}

if let Ok(n) = input_port
    .recv_one()
    .await
    .map(|v| GetData::<u64>::try_data(v).unwrap())
{
    /* n: u64 */
}
```

- `recv_many()` → `Result<TransmissionValue, ()>` — returns `Err` when the stream is closed.
- `recv_one()` → `Result<Value, ()>` — returns `Err` when the block is closed without a value.

### Sending data to outputs

```rust
// Send a single value
let _ = output_port.send_one(Value::U64(42)).await;
let _ = output_port.send_one(().into()).await;         // void
let _ = output_port.send_one(some_string.into()).await;

// Send many (stream)
check!(output_port.send_many(values).await)

// Close explicitly (usually not needed — the runtime handles it)
output_port.close().await;
```

`send_many` accepts a `TransmissionValue`. Common variants:
- `TransmissionValue::Other(VecDeque<Value>)` — heterogeneous batch
- `TransmissionValue::Byte(VecDeque<u8>)` — optimised for byte streams
- `TransmissionValue::Bool(VecDeque<bool>)` — optimised for bool streams

### The `check!` macro

`check!(expr)` is a short-circuit helper: if `expr` returns `Err`, it returns from the enclosing function immediately. Use it around every `.send_many()` call so that a closed downstream output terminates the treatment cleanly.

```rust
check!(output.send_many(batch).await)

// With a labelled block for early return inside a loop
'main: while ... {
    check!('main, output.send_one(val).await)
}
```

### Implicit variables available in the body

The macro injects additional local variables that are always available:

| Variable | Type | Description |
|----------|------|-------------|
| `track_id` | `TrackId` | Identifier of the current track |

### Full example — simple streaming treatment

```rust
/// Doubles every integer in a stream.
#[mel_treatment(
    generic N (Add)
    input value Stream<N>
    output doubled Stream<N>
)]
pub async fn double() {
    while let Ok(values) = value
        .recv_many()
        .await
        .map(|values| Into::<VecDeque<Value>>::into(values))
    {
        check!(
            doubled
                .send_many(TransmissionValue::Other(
                    values.into_iter().map(|v| v.add(v.clone())).collect()
                ))
                .await
        )
    }
}
```

### Full example — Block input and output

```rust
#[mel_treatment(
    generic T ()
    input value Block<T>
    output check Block<void>
)]
pub async fn check() {
    if let Ok(_) = value.recv_one().await {
        let _ = check.send_one(().into()).await;
    }
}
```

### Full example — model parameter

```rust
#[mel_treatment(
    model http_server HttpServer
    input trigger Block<void>
)]
pub async fn start() {
    let model = HttpServerModel::into(http_server);
    let http_server = model.inner();   // accesses the HttpServer struct

    if let Ok(_) = trigger.recv_one().await {
        http_server.launch_barrier.wait().await;
    }
}
```

The pattern `ModelTypeModel::into(name)` converts the injected model handle into the generated model wrapper, from which `.inner()` gives the actual Rust struct.

### Input content hints

An input can carry a content hint for documentation and tooling:

```rust
#[mel_treatment(
    input {content(json)} text Stream<string>
    output is_json Stream<bool>
)]
pub async fn validate() { ... }
```

The `{content(json)}` annotation tells Mélodium's tooling that the string has JSON content.

## `#[mel_function]` — defining a function

A function is a pure, synchronous `fn` (not `async`). It becomes a Mélodium function callable with the `|name()` syntax.

### Macro syntax

```rust
#[mel_function(
    generic T (TraitA + TraitB)
)]
pub fn function_name(arg: Type, other: OtherType) -> ReturnType {
    // pure computation
}
```

- No inputs/outputs — only parameters and a return value.
- The function must be deterministic and side-effect-free (it runs at program init or track creation, not during execution).
- Generic constraints follow the same syntax as `mel_treatment`.

### Examples

```rust
// No generics
#[mel_function]
pub fn version() -> string {
    melodium_engine::VERSION.to_string()
}

// With generic constraint
#[mel_function(
    generic T ()
)]
pub fn wrap(value: T) -> Option<T> {
    Some(value)
}

// With trait constraint
#[mel_function(
    generic N (Signed)
)]
pub fn abs(value: N) -> Option<N> {
    value.signed_abs()
}

// Returning a custom data type
#[mel_function]
pub fn ok() -> HttpStatus {
    HttpStatus(Status::Ok)
}
```

## `#[mel_data]` — defining a data type

A data type is a Rust struct or enum decorated with `#[mel_data]`. It must also derive `Debug` and `Clone` at minimum, plus whatever serialization traits are needed.

### Macro syntax

```rust
#[derive(Debug, Clone, PartialEq, Eq, Serialize, Deserialize)]
#[mel_data(traits(Trait1 Trait2 Trait3))]
pub struct MyType {
    pub field: SomeRustType,
}
```

The `traits(...)` list declares which Mélodium traits the type implements. The Rust implementations of those traits must also be present.

### Available Mélodium traits to declare

| Trait | What to implement in Rust |
|-------|--------------------------|
| `ToString` | `impl ToString for MyType { fn to_string(&self) -> string }` |
| `TryToString` | `fn try_to_string(&self) -> Option<string>` |
| `TryToBool` | `fn try_to_bool(&self) -> Option<bool>` |
| `TryToI64` | `fn try_to_i64(&self) -> Option<i64>` |
| `TryToU64` | `fn try_to_u64(&self) -> Option<u64>` |
| `TryToF64` | `fn try_to_f64(&self) -> Option<f64>` |
| `Display` | `fn display(&self, f: &mut core::fmt::Formatter<'_>) -> Result<(), core::fmt::Error>` |
| `Serialize` | derive `serde::Serialize` |
| `Deserialize` | derive `serde::Deserialize` |
| `PartialEquality` | derive `PartialEq` |
| `Equality` | derive `Eq` |
| `PartialOrder` | derive `PartialOrd` |
| `Order` | derive `Ord` |
| `Bounded` | provide `fn {name}_bounded_min() -> MyType` and `fn {name}_bounded_max() -> MyType` as free functions in the same module |

The `Display` trait here is Mélodium's own `Display` (from `melodium_core::executive`), not `std::fmt::Display`. It has the same method signature but lives in a different namespace.

### Example — simple data type

```rust
#[mel_data(traits(ToString PartialEquality Equality Display))]
#[derive(Debug, Clone, PartialEq, Eq, Serialize)]
pub struct HttpMethod(pub Method);

impl ToString for HttpMethod {
    fn to_string(&self) -> String {
        self.0.to_string()
    }
}

impl Display for HttpMethod {
    fn display(&self, f: &mut core::fmt::Formatter<'_>) -> Result<(), core::fmt::Error> {
        write!(f, "{}", melodium_core::executive::ToString::to_string(self))
    }
}
```

### Example — data type with Bounded

```rust
#[derive(Debug, Serialize, Deserialize, Copy, Clone, PartialEq, Eq, PartialOrd, Ord)]
#[mel_data(traits(Serialize Deserialize Bounded PartialEquality Equality PartialOrder Order))]
pub struct Level {
    pub level: LogLevel,
}

fn level_bounded_min() -> Level {
    Level { level: LogLevel::Trace }
}

fn level_bounded_max() -> Level {
    Level { level: LogLevel::Error }
}
```

The `bounded_min` / `bounded_max` free functions must be named `{snake_case_type_name}_bounded_min` and `{snake_case_type_name}_bounded_max`.

## `#[mel_model]` — defining a model

A model is a long-lived struct that the macro wraps into a generated `ModelNameModel` type. The struct must implement specific methods that the macro expects to find:

- `fn new(model: Weak<ModelNameModel>) -> Self` — constructor.
- `fn invoke_source(&self, source: &str, params: HashMap<String, Value>)` — called when a source is activated (e.g. when a `connection(...)` is created).
- Optionally `async fn continuous(&self)` — if declared in the macro with `continuous (method_name)`.
- Optionally `fn initialize(&self)` — if declared with `initialize method_name`.
- Optionally `fn shutdown(&self)` — if declared with `shutdown method_name`.

### Macro syntax

```rust
#[mel_model(
    param  param_name  Type  default_value        // model parameter; use `none` for required
    source source_name (ContextType) (            // source declaration
        param src_param SrcParamType none         //   parameters for this source
    ) (
        output_name Kind<Type>                    //   outputs produced by this source
    )
    continuous   (continuous_method)              // method called when the model starts
    initialize   initialize_method               // called once at initialization
    shutdown     shutdown_method                 // called on shutdown
)]
pub struct MyModel {
    model: Weak<MyModelModel>,
    // ... additional fields
}
```

- **`param name Type default`** — declares a model parameter. Use the literal `none` when the parameter has no default (required). Use the value itself for defaults (`10`, `0`, `true`, etc.).
- **`source name (ContextType) (source_params) (source_outputs)`** — a source that creates tracks. `ContextType` is the context type injected into created tracks (use `()` if none). The source outputs are what treatments in the created track can receive.
- **`continuous (fn_name)`** — runs `async fn fn_name(&self)` in the background for the lifetime of the model (e.g., an HTTP server loop).
- **`initialize fn_name`** — `fn fn_name(&self)` called once synchronously before the model starts (e.g., installing DB drivers).
- **`shutdown fn_name`** — `fn fn_name(&self)` called when the engine is shutting down.

### Accessing model parameters inside the model

The generated `ModelNameModel` wrapper exposes `get_param_name()` methods for each declared parameter:

```rust
let url: String = model.get_url();
let max: u32 = model.get_max_connections();
```

### Creating new tracks from a source

Inside the `continuous` method, call `model.new_source_name(context, &params, callback)` to spawn a new track:

```rust
model
    .new_ready(
        None,                          // optional parent TrackId
        &HashMap::new(),               // source parameters
        Some(Box::new(|mut outputs| {  // callback that returns the track's work
            let trigger = outputs.get("trigger");
            vec![Box::new(Box::pin(async move {
                let _ = trigger.send_one(().into()).await;
                trigger.close().await;
                ResultStatus::Ok
            }))]
        })),
    )
    .await;
```

The callback receives an `OutputSet` from which `outputs.get("port_name")` returns a `Box<dyn Output>`. The callback must return `Vec<Box<dyn Future<Output = ResultStatus>>>` — one future per concurrent task in the track.

### Example — simple model with one source

```rust
#[mel_model(
    source ready () () (trigger Block<void>)
    continuous (continuous)
)]
#[derive(Debug)]
pub struct Engine {
    model: std::sync::Weak<EngineModel>,
}

impl Engine {
    fn new(model: std::sync::Weak<EngineModel>) -> Self {
        Self { model }
    }

    async fn continuous(&self) {
        let model = self.model.upgrade().unwrap();
        model
            .new_ready(
                None,
                &HashMap::new(),
                Some(Box::new(|mut outputs| {
                    let trigger = outputs.get("trigger");
                    vec![Box::new(Box::pin(Self::ready(trigger)))]
                })),
            )
            .await;
    }

    async fn ready(trigger: Box<dyn Output>) -> ResultStatus {
        let _ = trigger.send_one(().into()).await;
        trigger.close().await;
        ResultStatus::Ok
    }

    fn invoke_source(&self, _source: &str, _params: HashMap<String, Value>) {}
}
```

### Example — model with parameters and initialize/shutdown

```rust
#[derive(Debug)]
#[mel_model(
    param url             string      none
    param max_connections u32         10
    param min_connections u32         0
    param acquire_timeout u64         10000
    param idle_timeout    Option<u64> 600000
    param max_lifetime    Option<u64> 1800000
    source connected () () (trigger Block<void>)
    source failure   () () (failed Block<void>  error Block<string>)
    source closed    () () (trigger Block<void>)
    initialize initialize
    shutdown   shutdown
)]
pub struct SqlPool {
    model: Weak<SqlPoolModel>,
    pool: AsyncRwLock<Option<AsyncArc<AnyPool>>>,
}

impl SqlPool {
    fn new(model: Weak<SqlPoolModel>) -> Self {
        Self { model, pool: AsyncRwLock::new(None) }
    }

    fn initialize(&self) {
        sqlx::any::install_default_drivers();
    }

    fn shutdown(&self) {
        // graceful teardown
    }

    fn invoke_source(&self, _source: &str, _params: HashMap<String, Value>) {}
}
```

### Example — model with context and source parameters

When a source creates tracks with a context, pass the context value to `new_source_name`:

```rust
#[mel_model(
    param host Ip   none
    param port u16  none
    source incoming (HttpRequest) (
        param method HttpMethod none
        param route  string     none
    ) (
        started   Block<void>
        headers   Block<StringMap>
        data      Stream<byte>
        finished  Block<void>
        completed Block<void>
        failed    Block<void>
        error     Block<string>
    )
    source failedBinding () () (
        failed Block<void>
        error  Block<string>
    )
    continuous (continuous)
    shutdown   shutdown
)]
pub struct HttpServer { ... }

impl HttpServer {
    async fn continuous(&self) {
        // ...
        model
            .new_incoming(
                None,
                http_request,   // the HttpRequest context value
                &params,        // HashMap<String, Value> with "method" and "route"
                Some(Box::new(move |mut outputs| {
                    let data = outputs.get("data");
                    // ... build track futures
                })),
            )
            .await;
    }
}
```

The `invoke_source` method is called when a Mélodium treatment using that source is declared; use it to register route handlers or similar setup:

```rust
fn invoke_source(&self, source: &str, params: HashMap<String, Value>) {
    match source {
        "incoming" => {
            let method = /* extract from params */;
            let route = /* extract from params */;
            self.routes.write().unwrap().push((method, route));
        }
        _ => {}
    }
}
```

## `#[mel_context]` — defining a context

A context is a plain Rust struct with public fields. The macro generates the Mélodium `@ContextName` element automatically.

### Macro syntax

```rust
#[mel_context]
pub struct MyContext {
    pub field_one: SomeType,
    pub field_two: OtherType,
}
```

No attributes inside the macro; just mark the struct fields `pub`. The field types must be valid Mélodium data types (core types or `#[mel_data]` types).

### Example

```rust
#[mel_context]
pub struct HttpRequest {
    pub id:         u128,
    pub route:      string,
    pub path:       string,
    pub parameters: StringMap,
    pub method:     HttpMethod,
}
```

This becomes `@HttpRequest` in Mélodium, with fields accessible as `@HttpRequest[id]`, `@HttpRequest[route]`, etc.

## Documentation comments

Doc comments on Rust items decorated with `mel_*` macros are forwarded to the Mélodium documentation system. Write them as standard `///` comments above the macro attribute, not above the `fn` or `struct` keyword directly — the macro must appear between the doc comment and the item.

```rust
/// Merge two streams.
///
/// The two streams are merged without predictable order.
///
/// ℹ️ Merge continues as long as `a` or `b` continues, while the other can be ended.
#[mel_treatment(
    generic T ()
    input a Stream<T>
    input b Stream<T>
    output value Stream<T>
)]
pub async fn merge() { ... }
```

Mermaid diagrams embedded in doc comments render in the generated HTML documentation:

```rust
/// ```mermaid
/// graph LR
///     T("emit(value=🟨)")
///     B["〈🟦〉"] -->|trigger| T
///     T -->|emit| S["〈🟨〉"]
/// ```
#[mel_treatment( ... )]
pub async fn emit(value: T) { ... }
```

## Type mapping between Rust and Mélodium

| Mélodium type | Rust type in function/treatment params | `Value` variant |
|---------------|----------------------------------------|-----------------|
| `void`        | `()` | `Value::Void(())` |
| `bool`        | `bool` | `Value::Bool(b)` |
| `byte`        | `u8` | `Value::Byte(n)` |
| `char`        | `char` | `Value::Char(c)` |
| `string`      | `String` (aliased as `string`) | `Value::String(s)` |
| `i8`…`i128`   | `i8`…`i128` | `Value::I8(n)` … |
| `u8`…`u128`   | `u8`…`u128` | `Value::U8(n)` … |
| `f32`, `f64`  | `f32`, `f64` | `Value::F32(n)`, `Value::F64(n)` |
| `Vec<T>`      | `Vec<Value>` / `VecDeque<Value>` | `Value::Vec(v)` |
| `Option<T>`   | `Option<Box<Value>>` | `Value::Option(opt)` |
| Custom data   | `Arc<dyn Data>` | `Value::Data(arc)` |

### Extracting typed values from `Value`

```rust
// Primitive
let n: u64 = GetData::<u64>::try_data(val).unwrap();
let s: String = GetData::<String>::try_data(val).unwrap();

// Custom data type — downcast the Arc
let obj: Arc<MyType> = GetData::<Arc<dyn Data>>::try_data(val)
    .unwrap()
    .downcast_arc::<MyType>()
    .unwrap();

// Vec of primitives via TryInto
let strings: Vec<String> = TryInto::<Vec<String>>::try_into(transmission_value).unwrap();
let bytes: Vec<u8> = TryInto::<Vec<u8>>::try_into(transmission_value).unwrap();
```

### Building `Value` from Rust

```rust
Value::U64(42)
Value::String("hello".to_string())
().into()                           // void
Some(42u64).into()                  // Option<u64>
Option::<String>::None.into()       // None
Value::Data(Arc::new(my_obj) as Arc<dyn Data>)
```

## Generic type parameters at runtime

Inside a treatment body, a generic type parameter `T` declared in `generic T (...)` is accessible as a value of type `DataType`. This lets you pass the runtime type to methods that need it:

```rust
#[mel_treatment(
    model concentrator Concentrator
    generic T ()
    input data Stream<T>
)]
pub async fn concentrate() {
    let model = ConcentratorModel::into(concentrator);
    let concentrator = model.inner();

    let data_type = T;   // DataType — the runtime type of T

    if let Some(sender) = concentrator.track_sender(track_id, data_type).await {
        while let Ok(value) = data.recv_one().await {
            check!(sender.send(value).await)
        }
    }
}
```

## Naming conventions summary

| Concept | Rust convention | Mélodium convention |
|---------|----------------|---------------------|
| Treatment / function name | `snake_case` | `camelCase` |
| Model name | `PascalCase` | `PascalCase` (unchanged) |
| Data type name | `PascalCase` | `PascalCase` (unchanged) |
| Context name | `PascalCase` | `@PascalCase` |
| Input / output port name | `snake_case` | `snake_case` (unchanged) |
| Treatment parameter | `snake_case` | `snake_case` (unchanged) |
| Model parameter | `snake_case` | `snake_case` (unchanged) |

The `camelCase` conversion applies only to the name of the `fn` itself (what becomes the Mélodium element name). Everything else keeps its Rust spelling.

## Patterns and idioms

### Streaming treatment that processes each value individually

```rust
while let Ok(values) = input
    .recv_many()
    .await
    .map(|v| Into::<VecDeque<Value>>::into(v))
{
    check!(
        output.send_many(TransmissionValue::Other(
            values.into_iter().map(|val| transform(val)).collect()
        )).await
    )
}
```

### Block treatment (receive once, send once)

```rust
if let Ok(val) = input.recv_one().await {
    let result = compute(val);
    let _ = output.send_one(result.into()).await;
}
```

### Concurrent sources with `futures::join!` / `select!`

```rust
use futures::{pin_mut, select, FutureExt};

let xa = async {
    while let Ok(a) = a.recv_many().await {
        check!(output.send_many(a).await);
    }
}.fuse();
let xb = async {
    while let Ok(b) = b.recv_many().await {
        check!(output.send_many(b).await);
    }
}.fuse();

pin_mut!(xa, xb);
loop {
    select! {
        () = xa => {},
        () = xb => {},
        complete => break,
    };
}
```

### Closing outputs explicitly before returning

When a Block output must be explicitly closed after sending (so downstream sees the port as done):

```rust
let _ = started.send_one(().into()).await;
started.close().await;
let _ = headers.send_one(headers_value).await;
headers.close().await;
```

### Accessing the model from a treatment

```rust
#[mel_treatment(
    model my_pool SqlPool
    input trigger Block<void>
)]
pub async fn connect() {
    let model = SqlPoolModel::into(my_pool);
    // model: Arc<SqlPoolModel>
    // model.inner(): &SqlPool
    // model.get_url(): String — from param
    if let Ok(_) = trigger.recv_one().await {
        model.inner().connect().await;
    }
}
```

The injected model variable (`my_pool` above) is consumed by `ModelTypeModel::into(...)` which returns an `Arc<ModelTypeModel>`. From the wrapper, `.inner()` yields a reference to the actual struct, and `.get_param_name()` yields parameter values.
