# Contributing to Mélodium

Thank you for your interest in contributing to **Mélodium**! We welcome contributions from everyone, whether you're fixing bugs, improving documentation, or adding new features.

---

## Code of Conduct

By participating in this project, you agree to abide by our [Code of Conduct](CODE_OF_CONDUCT.md). Please read it to understand the expectations for all contributors.

---

## How to Contribute

### 1. Reporting Bugs
If you encounter a bug, check the [work items](https://gitlab.com/melodium/melodium/-/work_items) to see if it has already been reported. If not, open a new issue using the provided template. Include:
- A clear description of the bug.
- Steps to reproduce it.
- Your environment (OS, Rust version, Mélodium version, etc.).

### 2. Suggesting Features
To propose a new feature, open an issue using the "Feature Request" template. Describe:
- The feature and its use case.
- Why it would be beneficial to the project.

### 3. Contributing Code
To contribute code:
1. **Fork** the repository on GitLab.
2. Clone your fork locally:
   ```bash
   git clone https://gitlab.com/[your-username]/[project].git
   ```
3. Create a new branch for your changes:
   ```bash
   git checkout -b [your-branch-name]
   ```
4. Make your changes and test them thoroughly.
5. Commit your changes with a descriptive message:
   ```bash
   git commit -m "Add [feature] or Fix [bug]"
   ```
6. Push your changes to your fork:
   ```bash
   git push origin [your-branch-name]
   ```
7. Open a **Merge Request (MR)** from your fork to the main repository. Use the MR template to describe your changes.

---

## Coding Standards

- Follow the project's coding style (e.g., idiomatic Rust with `cargo fmt` and Mélodium conventions).
- Ensure your code is well-documented.
- Write tests for your changes and ensure all tests pass before submitting an MR.

Ensure all CI tests pass before submitting an MR.
